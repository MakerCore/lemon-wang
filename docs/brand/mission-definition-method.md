**Lemon's MDM（Mission Definition Method）**

一套从技术核心出发、快速找到正确使命的决策框架。

适用场景：一套强大的技术，但最佳应用方向还不明确。市场信号模糊，赌错方向会烧掉大量资源。

核心逻辑：不从用户需求出发，而从技术的底层能力出发——提取核心使命、生成多个垂直场景、定义使命假设、用最小成本验证真实行为、做出保留/冻结/退出决策。

五个步骤：

1. **Extract** — 提取技术的核心使命：它能唯一做到什么？
2. **Generate** — 生成5-10个垂直场景，强制跨行业发散
3. **Define** — 写出一句使命假设："This technology exists to help [who] turn [X] into [Y], so they can [outcome]."
4. **Validate** — 用最便宜的MVP测试真实行为，预设kill criteria
5. **Decide** — 保留、冻结或退出。冻结不是放弃，技术资产继续流转

核心判断：使命对了，技术会自己长出产品。使命错了，再强的技术只是慢性烧钱。

---

**Lemon's Mission Tree**

MDM实际运行后生长出来的结构——同一个技术核心，向外扇形分裂出多个使命方向，每个方向对应不同的市场、用户心理、定价模型和验证结果。

它是MDM的可视化结果，也是决策记录。

Mission Tree系列文章是这个过程的公开档案：记录真实的技术核心、每个使命方向的验证过程、保留或冻结的决策依据。不是事后包装，是实时决策的第一手记录。

---

**两者关系：**

MDM是方法，Mission Tree是用这个方法做决策后长出来的结构。

一句话：**MDM是找使命的过程，Mission Tree是找到之后的样子。**
